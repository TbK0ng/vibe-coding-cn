# Vibe Coding 哲学原理

> 道生一，一生二，二生三，三生万物。

---

## 一：配置一个 AI CLI

**一**是起点，是你与 AI 世界的连接点。

安装一个 AI CLI（如 OpenCode），就是"道生一"——从无到有，获得了与 AI 对话的能力。

```bash
curl -fsSL https://opencode.ai/install | bash
```

这一行命令，就是你的"一"。

---

## 二：AI 操作一切文件

有了 AI CLI，你获得了**二**——读与写。

AI 可以：
- **读**：阅读任何文件、理解任何代码、分析任何结构
- **写**：创建文件、修改配置、生成代码

你不再需要手动编辑配置文件。告诉 AI 你要什么，它来操作。

```
帮我创建 .env 文件，配置数据库连接
```

```
阅读 package.json，安装所有依赖
```

---

## 三：AI 配置一切环境

**三**是 AI 的自我扩展能力。

AI 可以：
- 安装工具和依赖
- 配置开发环境
- 部署项目
- 安装 MCP 服务器扩展自身能力

```
帮我安装 Node.js、Python、Docker，配置好开发环境
```

```
克隆这个 GitHub 项目，阅读文档，完成所有配置
```

---

## 万物：AI 生成一切

当 AI 能读写文件、配置环境，它就能**生成万物**：

- 生成完整项目结构
- 生成数据库 Schema
- 生成 API 接口
- 生成前端页面
- 生成测试用例
- 生成部署脚本

你只需要描述**意图**，AI 负责**实现**。

---

## 核心心法

> **我是 AI 的寄生者，没有 AI 我就失去了一切能力。**

这不是自嘲，是觉悟。

- 不要手动编辑配置文件 → 让 AI 编辑
- 不要手动安装依赖 → 让 AI 安装
- 不要手动写样板代码 → 让 AI 生成
- 不要手动调试报错 → 把报错发给 AI

**你的工作是：描述意图、验证结果、做决策。**

**AI 的工作是：理解意图、执行操作、生成产出。**

---

## 实践

遇到任何问题？

1. 截图/复制错误信息
2. 描述你想要什么
3. 发给 AI
4. 让 AI 操作

不要自己动手，除非 AI 反复失败。
